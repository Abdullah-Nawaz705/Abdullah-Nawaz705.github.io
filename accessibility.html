Accessibility Statement

Last Updated: July 26, 2025

Abdullah Nawaz is committed to ensuring digital accessibility for people with disabilities. I am continually improving the user experience for everyone and applying the relevant accessibility standards.

Our Commitment

As a professional accessibility consultant, I believe that accessibility is not just a professional requirement but a fundamental right. This website has been designed and developed to meet or exceed WCAG 2.1 Level AA standards, with AAA implementation where feasible.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

This website is fully conformant with WCAG 2.1 Level AA. "Fully conformant" means that the content fully meets the accessibility standard without any exceptions.

Additionally, this website incorporates several Level AAA techniques where practical and beneficial to users.

Accessibility Features

This website includes the following accessibility features:

  • Keyboard Navigation: All interactive elements are accessible via keyboard navigation
  • Screen Reader Support: Comprehensive ARIA labeling and semantic markup
  • High Contrast: Color combinations meet or exceed WCAG AA contrast requirements
  • Responsive Design: Content adapts to different screen sizes and zoom levels up to 200%
  • Clear Navigation: Consistent navigation structure with skip links
  • Alternative Text: All images include appropriate alternative text descriptions
  • Focus Indicators: Clear visual focus indicators for keyboard users
  • Readable Content: Clear language, logical heading structure, and sufficient white space
  • Form Accessibility: Proper labels, error identification, and instructions
  • Reduced Motion: Respects user preferences for reduced motion

Testing Methods

This website has been tested using:

  • Manual Testing: Comprehensive keyboard navigation and screen reader testing
  • Assistive Technologies: Testing with NVDA, VoiceOver, and other screen readers
  • Automated Tools: Regular testing with accessibility scanning tools
  • Real Users: As a screen reader user myself, I regularly test all functionality
  • Code Review: Manual code review for semantic markup and ARIA implementation

Known Limitations

Despite our best efforts to ensure accessibility, there may be some limitations. Currently, we are not aware of any content that is not accessible. If you discover any accessibility barriers, please contact us immediately.
